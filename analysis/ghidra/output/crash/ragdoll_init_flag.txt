# Conclusion: The initialization order is:
#   1. Constructor: +0xa4=0, +0x2a4=0
#   2. Init step A: +0x2a4 = valid pointer (bone hierarchy)
#   3. Init step B: +0xa4 = allocated bone array (but entries=0)
#   4. Init step C: bone array entries filled with bone pointers
#   5. Skeleton update runs (FUN_00c79680)
#
# The crash: skeleton update runs between step 3 and step 4.
# The bone array exists but its entries are zero.
#

# with old bone pointers (zombie data). Step 5 reads valid-ish
# pointers even before step 4 runs. No crash.
