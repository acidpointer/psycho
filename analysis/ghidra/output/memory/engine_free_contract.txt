PART 3: MIMALLOC vs SBM BLOCK STATE AFTER FREE
======================================================================

mimalloc mi_free writes:
  offset 0: next-free pointer (4 bytes on 32-bit)
  All other bytes: UNTOUCHED

SBM FUN_00aa6c70 writes:
  (need to decompile to determine)

IF SBM also writes at offset 0 only, then mi_free is COMPATIBLE
with SBM's behavior for stale reads at offset 4+.

The CRASH comes from virtual calls: vtable ptr is at offset 0.
After mi_free, offset 0 = freelist next pointer (a heap address).
Reading this as vtable -> accessing freelist address as vtable ->
function pointer at vtable[N] is garbage -> EIP=0 crash.

With SBM: offset 0 = freelist next pointer (a POOL address).
Reading this as vtable -> accessing pool address as vtable ->
function pointer at vtable[N] is ALSO garbage.
BUT: SBM pool addresses are in game VA space (not zero/invalid).
The 'vtable' entries happen to be other pool blocks, which
contain non-zero data -> call goes to random game code, not 0x0.
This is STILL undefined behavior but doesn't crash with NULL.


PART 4: POTENTIAL ENGINE-LEVEL FIXES
======================================================================

Fix A: Hook the virtual dispatch in BST task processing.
  Before calling vtable[N], validate the vtable pointer.
  If vtable is in freelist range (not RDATA), skip the call.
  Requires: hook at the CALL instruction in BST loop.

Fix B: Hook NiSourceTexture destructor to cancel queued tasks.
  When NiSourceTexture is destroyed, find and cancel all
  QueuedTexture tasks that reference it.
  Requires: ability to walk the BST task queue.

Fix C: Replace mi_free's freelist write with a safe sentinel.
  Instead of writing freelist next ptr at offset 0, write a
  known 'dead vtable' pointer that has safe no-op methods.
  Requires: custom mimalloc modification OR post-free write.

Fix D: After mi_free, write a 'dead vtable' at offset 0.
  mi_free(ptr); *(uint32_t*)ptr = DEAD_VTABLE_ADDRESS;
  The dead vtable has all entries pointing to a no-op function.
  Stale reads get the dead vtable -> call no-op -> no crash.
  Requires: allocate a static vtable with NOP entries.
