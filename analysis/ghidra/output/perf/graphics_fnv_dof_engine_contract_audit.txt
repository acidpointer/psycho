1. How does ImageSpaceManager decide that effect ID 4 is active, and is its read-only active state stable after RenderEndOfFrameEffects?
2. Which exact +0x2A4..+0x2CD fields represent current/weather DOF strength, distance, range, mode, and mask?
3. Which gameplay systems write dialogue, iron-sight, VATS, and weather DOF state?
4. Can OMV detect native DOF without patching RenderEndOfFrameEffects or ImageSpaceManager::GetDepthTexture?
5. Does the existing post-ProcessImageSpaceShaders phase necessarily see vanilla DOF already composited?

Known source layout to verify, not assume:
  ImageSpaceManager +0x08: effect pointer array data; ID 4 is vanilla DOF
  ImageSpaceEffect vtable +0x18: IsActive; object +0x04: bIsActive candidate
  ImageSpaceManager +0x2A4..+0x2CD: current/weather DOF parameter candidates

Compatibility boundary:
  Keep 0x00B97900 RenderEndOfFrameEffects unpatched; Fallout Shader Loader replaces it.
  Keep 0x00B54090 GetDepthTexture and 0x00875E40 DepthResolve DOF site unpatched.
  Reuse OMV's existing ProcessImageSpaceShaders wrapper and independent world/first-person depth captures.
