FNV OMV PBR AND SCENE FOLLOWUP EVIDENCE PACK

This audit is deliberately bounded and non-recursive.
The large BDF790 selector is scanned linearly but is not decompiled.
Decompiler timeout per selected function: 20 seconds
Reference cap per target: 48
Call cap per function: 160

Questions covered:
1. What do the native sun camera/projector wrapper and pointer helper actually return?
2. How is the world current render target selected before OMV depth capture?
3. Which object/material data writes the six PPLighting texture arrays?
4. When is current NiD3DPass established relative to SetShaders and texture resolution?
5. Do selector, pass-entry, shader, and texture state belong to the same draw scope?
6. Which material offsets and resource records feed final texture-stage ownership?
7. Which terrain selector writes are distinct from general object shader pairs?
8. Which shader families and creation sites establish the native vertex/pixel ABI?
